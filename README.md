📧🚀 גשר אוטומציה ליצירת טיוטות ב-Outlook
פתרון Full-Stack שנועד לגשר על הפער בין אפליקציות Web לבין סביבת שולחן העבודה המקומית (Desktop). פרויקט זה מאפשר למשתמש לייצר מספר טיוטות אימייל ב-Outlook, כולל קבצים מצורפים, ישירות מתוך ממשק דפדפן.

📝 האתגר הטכנולוגי
דפדפני אינטרנט פועלים בסביבת Sandbox (ארגז חול), המונעת מהם אינטראקציה ישירה עם תוכנות מקומיות כמו Microsoft Outlook או גישה למערכת הקבצים המקומית מטעמי אבטחה. פתרונות סטנדרטיים כמו mailto: אינם תומכים בצירוף קבצים איכותי או ביצירה אוטומטית של מספר טיוטות נפרדות.

💡 הפתרון
הפרויקט מממש ארכיטקטורה של Local Agent (סוכן מקומי):

Frontend: ממשק HTML5/CSS3 מודרני ורספונסיבי להזנת נתונים.

Backend (הגשר): שרת Flask ב-Python הרץ מקומית ומתקשר עם ה-Outlook COM API באמצעות ספריית win32com.

ניהול פוקוס: שימוש ב-Windows API (win32gui) כדי לעקוף את מנגנון ה-"Focus Stealing Prevention" ולאלץ את חלון ה-Outlook לקפוץ לקדמת המסך.

🛠️ טכנולוגיות בשימוש
Frontend: HTML5, CSS3 (Flexbox, אנימציות), JavaScript (Fetch API).

Backend: Python 3.x, Flask.

אינטגרציה: PyWin32 (COM Objects), Windows API.

🚀 איך מפעילים את הפרויקט?
דרישות קדם
מערכת הפעלה Windows (נדרש עבור אינטגרציית ה-COM של Outlook).

Microsoft Outlook מותקן ומוגדר עם חשבון פעיל.

Python 3.x מותקן.

התקנה
שכפל את התיקייה למחשב שלך:

Bash
git clone https://github.com/your-username/outlook-draft-automation.git
התקן את הספריות הנדרשות:

Bash
pip install flask flask-cors pywin32
הרצה
הפעל את השרת המקומי:

Bash
python app.py
פתח את הקובץ index.html בדפדפן.

מלא את הפרטים, צרף קובץ ולחץ על "יצירת טיוטות ב-Outlook".

🧠 תכונות עיקריות ולוגיקה
תמיכה בריבוי נמענים: פיצול אוטומטי של רשימת מיילים המופרדים בפסיק לטיוטות נפרדות.

טיפול בקבצים מצורפים: שמירה זמנית של הקובץ שהועלה, צירופו ל-Outlook וביצוע "ניקוי" (Cleanup) של הקובץ מהשרת בסיום התהליך.

חווית משתמש (UI/UX): עיצוב נקי הכולל משוב ויזואלי למשתמש בזמן עיבוד הבקשה.

אוטומציה חכמה: המערכת בודקת אם Outlook רץ; אם לא, היא מפעילה אותו באופן אוטומטי לפני יצירת הטיוטות.
